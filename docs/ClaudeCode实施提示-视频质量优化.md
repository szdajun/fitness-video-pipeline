# Claude Code 实施提示：视频质量优化

## 目的

这份文档不是讲完整方案，而是给 Claude Code 直接落地时用的实施提示。重点围绕一类明确素材：

- 夜景
- 多人
- 广场/棚下健身
- 固定机位或较稳定机位
- 领操人在前排

对应当前样本目录：

- `C:\Users\18091\Desktop\短视频素材`

重点参考样本：

- `丽丽6.mp4`
- `艳青4.mp4`

---

## 总原则

这类视频优化目标不是：

- 更强美颜
- 更强塑形
- 更强运镜

而是：

1. 动作完整
2. 腿脚尽量不出框
3. 领操人稳定
4. 多人场景保留同步感
5. 夜景不过曝、不发灰、不脏

一句话：

**优先“稳、全、自然”，而不是“艳、瘦、推拉感”。**

---

## 改动优先级

### 第一优先级

1. 调整 H2V 裁切策略
2. 调整多人场景判定逻辑

### 第二优先级

3. 调整夜景调色策略
4. 降低多人远景中的 body warp 强度

### 第三优先级

5. 弱化 Ken Burns
6. 增加质量指标埋点

---

## 1. H2V 裁切：先保腿脚，再谈特写

### 当前问题

当前 H2V 对这类健身视频的最大风险不是“主体太小”，而是：

- 腿部空间不足
- 抬腿动作贴边
- 脚步出框
- 多人时后排下肢信息被吃掉

### 实施建议

优先检查并调整 [03_h2v_convert.py](/F:/wkspace/fitness-video-pipeline/stages/03_h2v_convert.py) 中这些点：

- 上部裁切比例
- 全景/特写判定阈值
- lead crop padding
- 多人场景下的宽画面保留比例

### 建议方向

- 不要再用过激的上部裁切
- `top_region_ratio` 往更保守方向调
- 推荐先试 `0.82 ~ 0.90`
- 多人场景优先保全身体完整性
- 横转竖时不要为了领操人脸更大而牺牲脚部动作

### 直接建议

如果当前代码仍有类似：

- `orig_h * 0.70`

这类固定值，优先把它改成更保守参数，并做成可配置。

---

## 2. 多人场景判定：不要太容易切成单人特写

### 当前问题

这类广场操素材的内容价值来自：

- 领操人
- 多人跟练
- 队形与同步感

如果人数稍少就切成很强的领操人 close-up，会损失群体氛围。

### 实施建议

优先调整：

- `pano_person_threshold`
- 横向铺开判定
- 全景/中景切换逻辑

### 建议方向

- 不要只按人数判断
- 增加多人横向分布宽度参考
- 只要多人铺开明显，就倾向保留更宽的画面
- 中景应比强特写更常见

### 实施目标

多人广场健身视频要更像“群体健身内容”，而不是“单人舞蹈特写”。

---

## 3. 夜景调色：控高光，轻提中暗部

### 当前问题

这类视频不是单纯太暗，而是：

- 顶灯和侧上方灯源明显
- 高光容易炸
- 暗部容易死黑
- 低码率下全局提亮会让画面更脏

### 实施建议

调整 [06_color_grade.py](/F:/wkspace/fitness-video-pipeline/stages/06_color_grade.py) 时，优先级建议是：

1. 压高光
2. 提中暗部
3. 饱和度轻调
4. CLAHE 保守

### 不建议

- 不要优先靠全局 brightness 提亮
- 不要在夜景里强推对比度和锐度
- 不要把灯光区域进一步推爆

### 后续可加但不是第一步

- highlight compression
- midtone lift
- scene-aware night mode

第一版可以先通过更保守的参数和顺序控制来改善。

---

## 4. Body warp：多人远景默认弱化

### 当前问题

在多人远景场景中，body warp 往往收益不大，但风险较高：

- 主体像素占比不够大
- 变形效果不明显
- 地面线条和邻近人容易被带歪
- 动作快时更容易露出痕迹

### 实施建议

对这类视频：

- 默认使用轻量 body warp
- 不要默认走强塑形 preset
- 当领操人占画面比例不足时，自动降级或关闭 body warp

### 推荐方向

- `overall_slim` 更保守
- `waist_slim` 更保守
- `leg_lengthen` 更保守
- 多人远景优先 `natural` 风格，而不是 `sexy`

---

## 5. Ken Burns：弱化，不做强运镜

### 当前问题

这些素材机位已经比较稳，所以主要需求不是人为增加镜头感。

过强 Ken Burns 会带来：

- 画面发飘
- 多人同步感下降
- 夜景低码率压缩痕迹更明显

### 实施建议

- 默认弱化 Ken Burns
- 即使用，也只做极轻微变化
- 优先做主体稳定，而不是额外推拉

### 结论

在这类视频上，“主体稳定跟随”比“镜头感增强”更重要。

---

## 6. 建议增加专用 preset

建议新增一个夜景多人健身 preset，例如：

- `night_square_dance`
- 或 `plaza_night_fitness`

### 这个 preset 的倾向应该是

- 保守 H2V 裁切
- 优先保腿部空间
- 多用中景/全景
- 低强度 body warp
- 控高光、轻提暗部
- 轻度降噪
- 弱锐化
- 默认关闭或弱化 Ken Burns

---

## 7. 建议优先埋点的指标

如果顺手做 metrics，优先埋这些：

- `ankle_out_of_frame_rate`
- `lead_center_jitter`
- `multi_person_width_ratio`
- `highlight_clipping_ratio`
- `body_warp_safe_ratio`

### 解释

- `ankle_out_of_frame_rate`
  这是健身视频里最重要的质量指标之一
- `lead_center_jitter`
  判断领操人跟随是否飘
- `multi_person_width_ratio`
  辅助判断是否该保留全景
- `highlight_clipping_ratio`
  判断夜景灯光是否过曝
- `body_warp_safe_ratio`
  判断 body warp 是否应该自动降级

---

## 推荐实施顺序

1. 先改 H2V 裁切参数和多人判定逻辑
2. 再调夜景调色策略
3. 再处理 body warp 降级
4. 再弱化 Ken Burns
5. 最后补 metrics 和 preset

原因：

- 当前最大观感风险是切脚、切腿、多人信息丢失
- 第二风险是夜景画面发灰、过曝、脏
- 体型优化和运镜不是这类素材的第一优先级

---

## 不建议当前优先做的事

- 不要先追求更强塑形
- 不要先追求更强美颜
- 不要先追求更强运镜
- 不要先为这类素材加过多“电影感”处理

对于广场健身视频，真实、完整、稳定，比“好看得很明显”更重要。

---

## 交付目标

Claude Code 完成这一轮优化后，理想效果应表现为：

- 领操人仍然清晰，但不会为了特写牺牲腿脚
- 多人场景保留足够同步氛围
- 夜景灯光不过曝
- 画面不发灰、不脏
- 后处理痕迹更轻
- 整体更像自然优化，而不是重加工视频
